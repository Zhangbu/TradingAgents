# Coding Standards

## 1. General Principles
- **Async First**: All I/O operations (API calls to exchanges, DB queries) must be `async`.
- **Type Safety**: Strict use of Python Type Hints and TypeScript interfaces.
- **Fail Fast**: Validate API keys and balance availability at startup.

## 2. Backend (Python/FastAPI)
- **Pattern**: Use the "Service Pattern". `routes.py` -> `services/` -> `execution/`.
- **Error Handling**: 
    - Never swallow exceptions in trading logic. 
    - Use custom exception classes (e.g., `ExchangeAPIError`, `InsufficientFundsError`).
- **Logging**: Use `structured logging`. Every agent thought process must be logged with a `session_id` and `timestamp`.
- **Environment**: Use Pydantic `BaseSettings` for `.env` management.

## 3. Frontend (React/TypeScript)
- **Component Structure**: Functional components using Tailwind CSS.
- **State Management**: Use `Lucide-react` for icons and `TanStack Query` (React Query) for API state.
- **Real-time**: Use a custom hook `useWebSocket` for handling agent log streams.

## 4. Trading Specifics
- **Currency Handling**: Use `Decimal` class for financial calculations, never `float`.
- **Idempotency**: Every order must have a unique `client_order_id` generated by our system to prevent double-spending on retries.
